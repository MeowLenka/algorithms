# Git, первое занятие

## Как понять, что происходит в репозитории?

В любой непонятной ситуации:

```bash
git status
```

Показывает:

- Какие файлы изменены
- Что готово к коммиту
- Что происходит с репозиторием (например, стадия merge)

## Как создать репозиторий?

Новый репозиторий в папке:

```bash
git init
```

Скачать существующий репозиторий:

```bash
git clone <url>
```

## Как сохранить изменения?

1. Добавить файлы в коммит

```bash
git add file.txt

git add dir/*.cpp dir/*.h
```

2. Создать локальный коммит

```bash
git commit -m "Обдуманное описание изменений"
```

3. Отправить коммит на сервер

```bash
git push
```  

## Как игнорировать файлы?

Создать файл `.gitignore` и перечислить в нём:

```
.vscode/
build/
*.log
```

## Как создать новую ветку и переключиться на неё?

```bash
git checkout -b new-branch
```

### Переключиться между ветками

```bash
git checkout branch-name
```

## Что делать, если не закончил работу, но нужно отвлечься?

Можно временно спрятать изменения

```bash
git stash 
```

И потом применить их

```bash
git stash pop
```
  
## Что делать, если закоммитил не то?

Последний коммит исчезнет, изменения останутся готовыми к новому коммиту:

```bash
git reset --soft HEAD~1
```

## Как полностью отменить изменения?

Следующей командой можно отменить все, что отмечено как `Changes not staged for commit` в выводе `git status`:
```bash
git reset --hard
```

⚠️ Удаляет изменения без возможности восстановления.
